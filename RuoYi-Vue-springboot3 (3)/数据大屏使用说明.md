# 学院科研数据大屏使用说明

## 📊 功能概述

数据大屏用于直观展示学院的各项科研统计数据，包括教师、学生、论文、专利、项目、资金等核心指标。

## 🎯 已实现功能

### 后端部分

1. **统计实体类** - `StatisticsView.java`
   - 位置：`ruoyi-admin/src/main/java/com/ruoyi/web/domain/StatisticsView.java`

2. **Mapper接口** - `DashboardMapper.java`
   - 位置：`ruoyi-admin/src/main/java/com/ruoyi/web/mapper/DashboardMapper.java`
   - 提供11个统计查询方法

3. **Mapper XML** - `DashboardMapper.xml`
   - 位置：`ruoyi-admin/src/main/resources/mapper/dashboard/DashboardMapper.xml`
   - 包含所有SQL统计查询

4. **Controller接口** - `DashboardController.java`
   - 位置：`ruoyi-admin/src/main/java/com/ruoyi/web/controller/dashboard/DashboardController.java`
   - 提供8个REST API接口：
     - `GET /dashboard/overview` - 总体概览数据
     - `GET /dashboard/faculty/title-distribution` - 教师职称分布
     - `GET /dashboard/student/status-distribution` - 学生学籍状态
     - `GET /dashboard/paper/yearly-trend` - 论文年度趋势
     - `GET /dashboard/funding/yearly-trend` - 资金年度趋势
     - `GET /dashboard/department/comparison` - 院系统计对比
     - `GET /dashboard/project/level-distribution` - 项目级别分布
     - `GET /dashboard/patent/type-distribution` - 专利类型分布
     - `POST /dashboard/statistics/refresh` - 手动刷新统计（需权限）

5. **定时任务** - `StatisticsRefreshTask.java`
   - 位置：`ruoyi-quartz/src/main/java/com/ruoyi/quartz/task/StatisticsRefreshTask.java`
   - Bean名称：`statisticsTask`
   - 提供3个方法：
     - `refreshGlobalStatistics()` - 刷新全局统计
     - `refreshDepartmentStatistics()` - 刷新院系统计
     - `refreshAllStatistics()` - 刷新所有统计

### 前端部分

1. **API封装** - `dashboard.js`
   - 位置：`ruoyi-ui/src/api/dashboard.js`

2. **数据大屏页面** - `DataScreen.vue`
   - 位置：`ruoyi-ui/src/views/dashboard/DataScreen.vue`
   - 包含7个统计卡片 + 7个图表

## 📋 配置步骤

### 1. 添加路由配置

编辑 `ruoyi-ui/src/router/index.js`，添加路由：

```javascript
{
  path: '/dashboard',
  component: Layout,
  hidden: false,
  children: [
    {
      path: 'data-screen',
      component: () => import('@/views/dashboard/DataScreen'),
      name: 'DataScreen',
      meta: { title: '数据大屏', icon: 'dashboard' }
    }
  ]
}
```

### 2. 配置定时任务（可选）

在若依系统的"定时任务"菜单中添加：

- **任务名称**：刷新统计数据
- **调用目标**：`statisticsTask.refreshAllStatistics`
- **cron表达式**：`0 0 2 * * ?` （每天凌晨2点执行）
- **执行策略**：立即执行

或者在数据库中直接插入：

```sql
INSERT INTO sys_job (job_name, job_group, invoke_target, cron_expression, misfire_policy, status, create_time) 
VALUES ('刷新统计数据', 'DEFAULT', 'statisticsTask.refreshAllStatistics', '0 0 2 * * ?', '3', '0', NOW());
```

### 3. 初始化统计数据

首次使用需要手动刷新一次数据：

**方式1：通过页面按钮**
- 访问数据大屏页面
- 点击右上角"刷新数据"按钮

**方式2：通过API调用**
```bash
POST /dashboard/statistics/refresh
```

**方式3：通过定时任务管理**
- 进入"系统监控 > 定时任务"
- 找到"刷新统计数据"任务
- 点击"执行一次"

## 🔧 技术栈

- **前端**：Vue 2 + Element UI + ECharts 5
- **后端**：Spring Boot 3 + MyBatis
- **数据库**：MySQL（STATISTICS_VIEW预聚合表）

## 📝 注意事项

1. **数据同步**
   - 统计数据来自 `STATISTICS_VIEW` 表
   - 需要定时任务或手动刷新来更新数据
   - 建议配置每天凌晨自动刷新

2. **权限控制**
   - 查看数据大屏：无需特殊权限
   - 手动刷新数据：需要 `dashboard:statistics:refresh` 权限

3. **性能优化**
   - 使用预聚合表，查询速度极快
   - 图表数据实时从业务表查询（分布和趋势数据）
   - 可根据需要也将分布数据加入预聚合表

4. **响应式布局**
   - 支持桌面端和移动端自适应
   - 图表会自动根据窗口大小调整
