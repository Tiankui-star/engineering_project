# 数据大屏配置和排查指南

## 🔍 问题排查清单

### ✅ 问题1：路由未配置（最可能的原因）

**现象**：访问不到数据大屏页面

**解决方案**：

#### 方法A：通过若依后台菜单管理配置（推荐）

1. 登录若依管理系统
2. 进入 **系统管理 → 菜单管理**
3. 点击 **新增** 按钮
4. 填写菜单信息：
   ```
   菜单名称：数据大屏
   上级菜单：系统工具（或根据需要选择）
   菜单类型：目录
   排序：10
   显示状态：显示
   路由名称：DataScreen
   组件路径：dashboard/DataScreen
   图标：dashboard
   权限标识：dashboard:view
   ```
5. 保存后，刷新页面即可看到菜单

#### 方法B：手动修改路由文件

编辑文件：`ruoyi-ui/src/router/index.js`

在适当位置添加路由：
```javascript
{
  path: '/dashboard',
  component: Layout,
  hidden: false,
  children: [
    {
      path: 'data-screen',
      component: () => import('@/views/dashboard/DataScreen'),
      name: 'DataScreen',
      meta: { title: '数据大屏', icon: 'dashboard' }
    }
  ]
}
```

### ✅ 问题2：ECharts未安装

**现象**：页面空白，控制台报错 `Cannot find module 'echarts'`

**解决方案**：

```bash
cd ruoyi-ui
npm install echarts@5 --save
```

或者使用yarn：
```bash
cd ruoyi-ui
yarn add echarts@5
```

---

### ✅ 问题3：后端接口未响应

**检查方法**：

1. 打开浏览器开发者工具（F12）
2. 切换到 **Network** 标签
3. 访问数据大屏页面
4. 查看是否有 `/dashboard/overview` 等请求
5. 检查响应状态码

**可能的错误**：

#### 错误A：404 Not Found
- **原因**：后端Controller未被扫描
- **解决**：检查 `RuoYiApplication.java` 的包扫描路径

#### 错误B：401 Unauthorized
- **原因**：未登录或权限不足
- **解决**：先登录系统，或临时移除权限验证

#### 错误C：500 Internal Server Error
- **原因**：SQL执行错误或数据库连接问题
- **解决**：查看后端日志

---

### ✅ 问题4：数据库表无数据

**检查方法**：

```sql
-- 检查统计数据
SELECT * FROM STATISTICS_VIEW WHERE dimension_type = 'GLOBAL';

-- 应该看到7条记录：
-- total_faculty, total_student, total_paper, 
-- total_patent, total_project, total_award, total_funding
```

**如果没数据**：

执行初始化脚本：
```bash
mysql -u root -p your_database < sql/init_dashboard_statistics.sql
```

---

### ✅ 问题5：Mapper未被扫描

**检查方法**：

查看后端启动日志，搜索：
```
DashboardMapper
```

**如果未找到**：

在 `application.yml` 中确认Mapper扫描路径：
```yaml
mybatis:
  mapperLocations: classpath*:mapper/**/*Mapper.xml
```

## 🚀 完整的配置步骤

### 步骤1：确认数据库

```sql
-- 1. 检查表是否存在
SHOW TABLES LIKE 'STATISTICS_VIEW';

-- 2. 检查数据是否存在
SELECT * FROM STATISTICS_VIEW;

-- 3. 如果没有数据，执行初始化
-- 使用 sql/init_dashboard_statistics.sql
```

### 步骤2：确认后端

1. 检查文件是否存在：
   ```
   ✅ ruoyi-admin/src/main/java/com/ruoyi/web/controller/dashboard/DashboardController.java
   ✅ ruoyi-admin/src/main/java/com/ruoyi/web/mapper/DashboardMapper.java
   ✅ ruoyi-admin/src/main/java/com/ruoyi/web/domain/StatisticsView.java
   ✅ ruoyi-admin/src/main/resources/mapper/dashboard/DashboardMapper.xml
   ✅ ruoyi-admin/src/main/java/com/ruoyi/web/task/StatisticsRefreshTask.java
   ```

2. 重启后端服务

3. 测试接口：
   ```
   http://localhost:8080/dashboard/overview
   ```

### 步骤3：确认前端

1. 检查文件是否存在：
   ```
   ✅ ruoyi-ui/src/views/dashboard/DataScreen.vue
   ✅ ruoyi-ui/src/api/dashboard.js
   ```

2. 安装ECharts：
   ```bash
   cd ruoyi-ui
   npm install echarts@5
   ```

3. 配置路由（见上面的方法A或B）

4. 重启前端服务：
   ```bash
   npm run dev
   ```

### 步骤4：配置菜单权限

1. 登录若依后台
2. 系统管理 → 菜单管理 → 新增菜单
3. 填写：
   - 菜单名称：数据大屏
   - 路由地址：dashboard
   - 组件路径：dashboard/DataScreen
   - 权限标识：dashboard:view

4. 系统管理 → 角色管理 → 分配权限
5. 给相应角色添加"数据大屏"菜单权限

### 步骤5：测试

1. 刷新浏览器
2. 在左侧菜单找到"数据大屏"
3. 点击进入
4. 应该能看到数据和图表

## 🔧 常见错误和解决方案

### 错误1：页面空白，无报错

**原因**：路由未配置或组件路径错误

**解决**：
- 检查路由配置
- 确认组件路径是 `dashboard/DataScreen`（不是 `@/views/...`）

### 错误2：控制台报错 `Cannot read property 'totalFaculty' of undefined`

**原因**：API返回数据结构不对

**解决**：
- 检查后端返回格式
- 确认是 `{ code: 200, data: {...} }` 格式

### 错误3：图表不显示

**原因**：ECharts未安装或未正确引入

**解决**：
```bash
npm install echarts@5
```

检查 DataScreen.vue 中的引入：
```javascript
import * as echarts from 'echarts'
```

### 错误4：刷新按钮报错 403

**原因**：权限不足

**解决**：
- 临时方案：移除 `@PreAuthorize` 注解
- 正式方案：在菜单管理中添加权限 `dashboard:statistics:refresh`

## 📝 快速诊断脚本

在浏览器控制台执行：

```javascript
// 测试API是否可用
fetch('/dashboard/overview')
  .then(res => res.json())
  .then(data => {
    console.log('API响应:', data)
    if (data.code === 200) {
      console.log('✅ 后端接口正常')
      console.log('数据:', data.data)
    } else {
      console.error('❌ 后端接口异常:', data.msg)
    }
  })
  .catch(err => {
    console.error('❌ 网络请求失败:', err)
  })
```
